# Governance

## Purpose

Governance documents define the rules, structure, and permissions that guide design system usage and application behavior. This includes information architecture, permission matrices, and design tool organization.

## Files in This Category

- **`information-architecture.md`** - Complete 8-domain app sitemap, Figma organization, governance rules (consolidated from 2 files)
- **`permissions-matrix.md`** - Role-based access control matrix, permission levels by domain (extracted from navigation pattern)
- **`figma-organization.md`** - Design file and component organization, workspace structure (moved from components)

## When to Use

Reference these files when:
- Understanding application structure and navigation hierarchy
- Checking user permissions and role capabilities
- Organizing design files and components
- Planning access control for new features
