# Screens

## Purpose

Screens contain feature-specific implementations that show how patterns and components come together to create complete user-facing features. Each screen spec includes layout, behavior, and design details.

## Files in This Category

- **`dashboards.md`** - Founder (KPIs, cash flow), Accountant (verification, reconciliation), Individual (widgets) - consolidated from 3 files
- **`transactions.md`** - Transaction list, categorization, reconciliation, bulk actions, expandable details - extracted from table pattern
- **`reconciliation.md`** - Bank reconciliation workflow, matching interface, difference handling
- **`invoicing.md`** - Invoice creation, payment tracking, aging reports, credit notes
- **`reports.md`** - P&L, balance sheet, cash flow, AR/AP aging, drill-down
- **`budgets-goals.md`** - Budget creation, variance analysis, savings goals, milestones
- **`assets.md`** - Asset register, depreciation schedules, capitalization, disposals
- **`filing.md`** - Tax filing readiness, document collection, deadline tracking, jurisdiction compliance

## When to Use

Reference these files when:
- Implementing feature screens
- Understanding feature-specific design requirements
- Reviewing screen layouts and interactions
- Following screen specifications in development
