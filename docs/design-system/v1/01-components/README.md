# Components

## Purpose

Components are reusable UI building blocks that apply design foundations to solve specific UI problems. Each component should be self-contained, documented, and ready for use across the application.

## Files in This Category

- **`primitives.md`** - Buttons, inputs, badges, chips, icons (basic building blocks)
- **`data-display.md`** - Cards, tables, lists, accordions, loading states, empty states
- **`feedback.md`** - Toasts, inline alerts, confirmations, validation, progress indicators
- **`financial-components.md`** - Money amounts, GL selectors, entity badges, KPI cards, journal previews
- **`ai-components.md`** - Insight cards, suggestion chips, confidence badges, critical alerts, escalation

## When to Use

Reference these files when:
- Building new UI elements
- Ensuring component consistency across the application
- Looking for reusable patterns to avoid duplication
- Understanding component behavior and states
