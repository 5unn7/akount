# Compliance

## Purpose

Compliance documents specify regulatory requirements, security standards, and SOC 2 controls that the application must satisfy. These requirements inform design decisions and implementation constraints.

## Files in This Category

- **`soc2.md`** - SOC 2 Type II controls, audit trails, data retention, access controls, risk management
- **`security.md`** - Authentication, authorization, encryption, input validation, sensitive data handling
- **`regulatory-requirements.md`** - Multi-jurisdiction tax compliance, financial reporting, cross-border rules

## When to Use

Reference these files when:
- Implementing features that handle sensitive data
- Designing security and permission controls
- Documenting audit trails
- Planning compliance-related features
- Reviewing data handling practices
