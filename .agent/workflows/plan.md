---

1. Create implementation plan
Create a new file in `docs/plans/YYYY-MM-DD-feature-name-plan.md`.
2. Define the goal
Write a clear objective for the change.
3. List proposed changes
Group changes by component (api, web, shared). Use [NEW], [MODIFY], [DELETE] labels.
4. Enforcement of standards
Explicitly describe how Multi-Tenancy and financial standards will be maintained.
5. Create a verification plan
List automated tests and manual verification steps.
6. Request user review
Ask the user to approve the plan before starting work.
