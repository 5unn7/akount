1. Create a brainstorm document
Create a new file in documentation directory `docs/brainstorms/YYYY-MM-DD-feature-name-brainstorm.md`.
2. Analyze the prompt and research existing code
Research the codebase for similar patterns and dependencies.
3. Formulate key questions
Identify ambiguities and ask the user for clarification.
4. Draft technical approaches
Outline 2-3 approaches with tradeoffs and multi-tenant implications.
5. Review with the user
Present the brainstorm and wait for feedback before proceeding to planning.
