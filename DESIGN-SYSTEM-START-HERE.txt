      const pathname = usePathname();
      // Active state logic here
  }
  WHY: usePathname requires 'use client' - this is correct

PATTERN 3: Hydration-Safe Theme Toggle
  'use client';
  const [mounted, setMounted] = useState(false);
  useEffect(() => setMounted(true), []);
  if (!mounted) return <Button disabled />;
  // Now safe to access localStorage
  WHY: Prevents hydration mismatches when accessing browser APIs

REMEMBER
================================================================================

1. Server Component by Default
   Pages and layouts should be Server Components
   Only use 'use client' when you need hooks, events, or browser APIs

2. Minimal Client Boundaries
   Extract interactive parts to Client Components
   Pass data from server to client via props

3. CSS Variables for Theming
   Use --primary, --secondary, etc.
   Theme switching happens via dark class on html element

4. Hydration Safety
   When accessing localStorage or browser APIs:
   1. Create mounted state
   2. Check mounted in useEffect
   3. Return fallback before mounted
   4. Access browser APIs only after mounted

5. Follow the Patterns
   All new components should follow the documented patterns
   You have 8 reference patterns to choose from
